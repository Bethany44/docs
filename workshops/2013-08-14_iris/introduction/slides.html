What does it do?

Provide complete framework to work with any type seismological data in Python

  • read/write file format support
    • waveforms (MiniSEED, SAC, GSE, SEISAN, SEG-Y, ...)
    • metadata (SEED, QuakeML, StationXML)
      •
  • data center access
    • IRIS webservices, ArcLink, Neries/Nera, Earthworm, NEIC, Seishub, SeedLink
      •
  • unified object classes
  • basic signal processing routines
    • trimming, merging, tapering, rotating, resampling, ...
    • filtering, instrument correction, ...
    • beamforming, cpx trace analysis, CC routines, PPSD, triggers, ...
      •

What's the beauty of it?

  • Runs on all Platforms
    • Windows, Mac, Linux, FreeBSD (Installers, Debian/Ubuntu repository, ...)
    • from RaspberryPi (35$) to SuperMUC (>150 Mio.$)
      •
  • Free and open source (LGPL/GPL)
  • Community pages: GitHub, wiki, tutorials, installation instructions, ticket system, mailing list
  • Well documented source code, auto-generated API docs
  • Test-driven development (a.t.m. more than 900 unit tests)
  • Reuses well established third-party code
    • Python community: numpy, scipy, matplotlib, ...
    • Seismo community: libmseed, GSE UTI, ...
      •
  • Python!

Python!

  • One of the most used programming languages
  • Easy to learn, very concise and easy-to-read syntax
  • Free and Open Source, large community
  • Cross-platform: Runs everywhere from RaspberryPi to large supercomputers
  • General purpose language (in contrast to many other tools used in science)
  • No need to compile and interactive shell available
  • "Batteries included"
  • Mature third party libraries
  • Easy to interact with existing C and Fortran code

Do people actually use it?

  • ~250 people on users mailing list
  • 30+ scientific citations (thanks guys!)

Who's behind it?

  • since then around 20-25 people from all around the world contributed
  • 4000+ code commits
  • 600+ tickets and pull requests
